```yaml

  # Firebase
  firebase_core: ^3.6.0
  firebase_auth: ^5.3.1
  cloud_firestore: ^5.4.4
  cloud_functions: ^5.1.3

  # Stripe
  flutter_stripe: ^10.2.0

  # URL launcher (for Stripe Connect onboarding)
  url_launcher: ^6.3.0

  # Date formatting (used in payment_screen.dart)
  intl: ^0.19.0
```

Then run:
```bash
flutter pub get
```

## 3. iOS — Info.plist changes

Open `ios/Runner/Info.plist` and add inside the root `<dict>`:

```xml
<!-- Stripe URL scheme for 3DS redirects -->
<key>LSApplicationQueriesSchemes</key>
<array>
  <string>stripe</string>
</array>

<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array>
      <string>refreshme</string>
    </array>
  </dict>
</array>
```

## 6. Navigate to PaymentScreen when client books

```dart
import '../../screens/booking/payment_screen.dart';

// When client taps "Book Now":
Navigator.push(
  context,
  MaterialPageRoute(
    builder: (_) => PaymentScreen(
      stylistId: stylist.id,
      stylistName: stylist.name,
      stylistPhotoUrl: stylist.profileImageUrl ?? '',
      stylistStripeAccountId: stylist.stripeAccountId,
      serviceName: selectedService.name,
      servicePrice: selectedService.price,
      bookingDate: selectedDateTime,
      isMobile: isMobileAppointment,
    ),
  ),
);
```

## How payments work (reminder)

```
Client books → PaymentScreen → Stripe PaymentSheet (20% deposit)
   → Firebase Function splits: 90% to stylist, 10% to RefreshMe
   → Stylist sees payout in their Stripe Express dashboard
```

No subscription needed — RefreshMe earns its cut automatically from every booking.
